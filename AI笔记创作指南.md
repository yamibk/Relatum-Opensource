# AI 笔记创作指南 — 用画布生成 / 整理 / 美化笔记

> **这份给谁看**：被指派「帮用户生成笔记、整理笔记、排版美化、做思维导图」的 AI agent。
> **这份不负责什么**：不讲怎么修改画布**程序本身**（那是 `AGENTS.md` 的事，本文件读者无需读它）。外部交付时，你的产出是**笔记数据**（`.canvas` 文件或一组 `.md` 文件），不是改源码。
> **一句话**：外部交付时，你只要会写两种文本文件，就能让用户在画布里得到漂亮的笔记。本文件也会被内置 AI 助手当作背景指南；若后端额外给出“只输出 JSON”的契约，以后端契约为准。

---

## 0. 30 秒速览（先读这段）

- 画布的每张笔记 = 一个 **`.canvas` 文件**（JSON：`nodes` 节点 + `edges` 连线）。
- 外部交付有**两条路径**，按任务选：
  1. **写一组 `.md` 文件夹** → 用户「导入」→ 画布**自动布局+自动配色**。**做多篇互联笔记/知识库最快**，你不用算坐标。
  2. **直接写 `.canvas` 文件** → 用户「打开」。**全控制**：能用全部节点类型、精确排版、图形装饰、思维导图。
- 笔记正文是 **Markdown**，而且支持一堆增强语法：高光 / 字色 / 字号 / Callout / 推导链 / 双链 / 公式 / 表格（见 §4）。
- **视觉红线**：UI 强调色禁止蓝色；但**笔记内容里的颜色标签可以用蓝**（那是内容色，不是 UI 色）。配色要克制、服务信息层级。
- 交付后用一句话告诉用户怎么打开（见 §8）。
- **特殊情况**：如果你是画布内置 AI 助手的一次“生成到画布”请求，不要写 `.md` 文件夹或 `.canvas` 文件；按后端追加的输出契约返回纯 JSON，由后端布局、前端预览后注入当前画布。

## 1. 外部交付路径，怎么选

| 任务 | 推荐路径 | 为什么 |
|---|---|---|
| 一次生成**很多篇**互相关联的笔记（知识库、课程章节群、概念网络） | **路径①：`.md` 文件夹导入** | 后端自动力导向布局+按簇配色，你只写内容和 `[[链接]]`，零坐标计算 |
| 单张结构化笔记、要用**索引/卡片/代码/便签**多种节点、要**图形装饰/色块分区/精确排版** | **路径②：直接写 `.canvas`** | 只有这条能完整控制节点类型、坐标、装饰、样式 |
| 思维导图 | 两条都行（见 §6） | 写节点+父子连线，导入或打开后由用户点「脑图整理」自动成形 |
| 「美化/整理」用户**已有**的画布 | **路径②**：读入他的 `.canvas`，重排坐标/换节点类型/补色块/统一样式后写回 | 见 §7 |

> 拿不准时：**内容多、关系网状 → 路径①**；**要精雕单页 → 路径②**。

### 1.1 内置 AI 助手特殊场景（不写文件）

画布里右侧的内置 AI 助手会把本指南作为背景说明，但后端会在本指南后面追加更高优先级的输出契约。处在这个场景时：

- **不要**按 §8 交付 `.md` 文件夹或 `.canvas` 文件，也不要输出解释文字或代码块。
- 只按后端契约输出一个 JSON 对象，通常包含 `nodes` 与 `edges`；节点可用 `card/index/preview/sticky/code`，连线端点按后端契约写新节点下标或已有节点 id。
- 坐标和布局由后端处理，前端会先给用户预览，用户确认后才注入当前画布；不满意可以整批撤销。
- 如果没有看到后端契约，就按本文件的两条外部交付路径执行。

## 2. 路径①：写一组 `.md` 文件夹（自动成图）

最省力的批量方式。**画布会把一个文件夹里的 `.md` 自动变成一张画布**。

### 规则（务必遵守，否则导入报错）

1. 一个**扁平文件夹**，里面只放 `.md` 文件（**不能有子文件夹或别的类型文件**，否则整体拒绝导入）。
2. **每个 `.md` = 一张卡片节点**：
   - **文件名（去掉 `.md`）= 节点标题**。所以文件名要正是你想显示的标题。
   - **文件正文 = 节点正文 `body`**（支持 §4 全部 Markdown 语法）。
3. **节点之间的连线 = 双链**：在某个 `.md` 文件的**最开头**，逐行写 `[[目标文件名]]`（一行一个，不含 `.md` 后缀），就会和那些文件连线。
   - `[[名字]]` 必须**精确匹配另一个文件的文件名**（大小写不敏感），匹配不到会报错。
   - 这些 `[[…]]` 行必须在**文件最顶部、连续**；遇到第一行非 `[[…]]` 就停止，其余全算正文。建议链接行后空一行再写正文。
   - 连线是无向的（A 链接 B，A、B 之间就有一条线），不重复连。
4. 文件名不能重复（大小写不敏感）；不能 `[[自己]]`。
5. 文件用 **UTF-8** 编码（带不带 BOM 都行）。

### 导入后画布自动做的事（你不用管）

- **力导向布局**：互相链接的笔记抱成一簇、自动散开收敛；多个簇货架式铺开、不重叠。**完全确定性**，同一组文件每次导入位置一致。
- **自动配色**：每个「成簇」（≥2 篇互联）的群组分配一种柔和色（蓝/绿/黄/红/紫轮流）；孤立笔记保持中性。
- 连线统一为柔和灰、smooth 曲线。
- 导入的节点**一律是卡片节点**（`kind:"card"`）。如果你需要索引/代码/便签等其它类型，请改用路径②。

### 示例：3 个文件的小知识库

**`极限.md`**
```markdown
[[导数]]
[[连续性]]

极限是微积分的基石。当 $x \to a$ 时 $f(x)$ 无限接近某个值 $L$，记作 $\lim_{x\to a} f(x)=L$。

> [!tip] 直觉
> 不是"到达"，而是"要多近有多近"。
```

**`导数.md`**
```markdown
[[极限]]

导数是函数在某点的瞬时变化率，本质是一个**极限**：

$$f'(x)=\lim_{h\to 0}\frac{f(x+h)-f(x)}{h}$$
```

**`连续性.md`**
```markdown
[[极限]]

函数在某点连续 $\iff$ 该点的==极限值==等于==函数值==。
```

用户把这个文件夹一导入，就得到一张「极限」居中、向「导数」「连续性」放射、自动上色的画布。

## 3. 路径②：直接写 `.canvas` 文件

全控制路径。`.canvas` 是一个 **JSON 文件**，UTF-8 编码。

### 3.1 最小骨架

```json
{
  "version": 1,
  "createdAt": "2026-06-03T10:00:00",
  "updatedAt": "2026-06-03T10:00:00",
  "nodes": [],
  "edges": []
}
```

- `version` 固定 `1`。`createdAt`/`updatedAt` 用 ISO 时间（`YYYY-MM-DDTHH:MM:SS`，不带毫秒）；保存时画布会自动刷新 `updatedAt`，你写个合理值即可。
- 真正的内容在 `nodes`（节点数组）和 `edges`（连线数组）。

### 3.2 id 规则

- 每个节点要有**唯一** `id`，每条连线也要。**只要全文件唯一就行**，加载器不校验格式。
- 约定（推荐照用，便于阅读）：节点 `n_xxx`、连线 `e_xxx`。批量生成时用 `n_1 / n_2 / …`、`e_1 / e_2 / …` 这种简单唯一串完全可以。
- **连线的 `from` / `to` 必须填某个真实存在的节点 `id`**，否则那条线连不上。

### 3.3 坐标系（重要）

- 单位是**画布像素**；`x` / `y` 是节点的**左上角**坐标。可以是负数（画布是无限平移的）。
- 横向 `x` 向右增大，纵向 `y` 向下增大（屏幕坐标系）。
- **正文节点（卡片/索引/预览/便签/代码）不要写 `width`/`height`** —— 它们按内容自动撑开。手动写尺寸反而会打架。
- **只有装饰节点**（图形 `shape` / 图片 `image` / 附件 `pdf`/`md`）**才需要 `width`/`height`**。
- 布局留白参考：相邻节点中心至少隔 ~**320px 横向 / ~200px 纵向**，笔记不会挤在一起（这也是导入自动布局用的间距基准）。

### 3.4 节点类型速查

新建笔记时**首选卡片节点**；需要目录/悬停/灵感/代码时再用其它类型。

| `kind` | 名字 | 用途 | 画布上长什么样 | 关键字段 |
|---|---|---|---|---|
| `"card"` | 卡片 | **默认主力**，承载一段正文知识 | 标题+正文**常驻**显示 | `text` 标题、`body` 正文 |
| `"index"` | 索引 | 章节目录/导航中枢 | 标题 + **沿连线自动生成只读目录**；自己也可有正文 | `text`、`body`、`indexDepth`(1–6,默认4) |
| `"preview"` | 预览 | 折叠的细节，平时收着 | 只显标题，**悬停才展开正文** | `text`、`body` |
| `"sticky"` | 便签 | 灵感/旁注/提醒 | 小张正文便签，自带柔和底色 | `text`、`body`、`bgColor`(可省=随机) |
| `"code"` | 代码 | 记 C/Python/MATLAB 源码 | 代码块、不解析 Markdown/公式 | `text`(自动取首行)、`body`(源码)、`language` |

**通用可选样式字段**（缺省=朴素默认，按需才加）：

| 字段 | 含义 | 取值 |
|---|---|---|
| `color` | 节点配色（命名色） | `gray`/`blue`/`green`/`yellow`/`red`/`purple` |
| `bgColor` | 自定义底色 | 十六进制如 `#ebf1fb`（与 `color` 二选一即可） |
| `borderColor` | 边框色 | 十六进制 |
| `scale` | 缩放 | 数字，如 `1.2` |
| `mindmapCollapsed` | 收起其脑图后代 | `true` |

> **配色用法**：颜色是用来**编码含义**的（比如红=重点、蓝=定义、绿=例子），不是越花越好。一张笔记里**别超过 3~4 种**节点色。命名色 `gray/blue/green/yellow/red/purple` 对应柔和底色 `#f2f2f1 / #ebf1fb / #eaf5ec / #fbf4e2 / #fceceb / #f2ecfb`。

#### 节点示例

```json
{ "id": "n_1", "x": 0,   "y": 0,   "kind": "index", "text": "微积分总览", "indexDepth": 3 },
{ "id": "n_2", "x": 360, "y": -80, "kind": "card",  "text": "极限",
  "body": "极限是微积分的基石。\n\n$$\\lim_{x\\to a} f(x)=L$$", "color": "blue" },
{ "id": "n_3", "x": 360, "y": 120, "kind": "preview", "text": "ε–δ 定义",
  "body": "对任意 $\\varepsilon>0$，存在 $\\delta>0$……" },
{ "id": "n_4", "x": 720, "y": 0,   "kind": "code", "text": "梯形积分",
  "body": "def trapz(f, a, b, n):\n    h = (b-a)/n\n    return h*(sum(f(a+i*h) for i in range(1,n)) + (f(a)+f(b))/2)",
  "language": "python" },
{ "id": "n_5", "x": 60,  "y": 220, "kind": "sticky", "text": "待办",
  "body": "复习中值定理证明" }
```

> JSON 里正文 `body` 的换行写成 `\n`，反斜杠（公式里很多）写成 `\\`。这是 JSON 转义，不是画布的要求。

### 3.5 连线 `edges`

```json
{ "id": "e_1", "from": "n_1", "to": "n_2", "text": "包含",
  "curve": "smooth", "color": "#bcbcbc", "arrow": "to", "lineStyle": "solid" }
```

| 字段 | 含义 | 取值（缺省=默认） |
|---|---|---|
| `from` / `to` | 起点 / 终点节点 id（**方向 = 父→子**，见下） | 必填，真实节点 id |
| `text` | 线上标签 | 字符串，可空 |
| `curve` | 线型 | `bezier`/`straight`/`elbow`/`rounded-elbow`/`s-curve`/`smooth` |
| `color` | 颜色 | 十六进制，如柔和灰 `#bcbcbc` |
| `arrow` | 箭头 | `none`/`to`/`from`/`both` |
| `lineStyle` | 实虚 | `solid`/`dashed` |
| `width` | 线宽 | 数字 |

- **方向语义很关键**：`from→to` 被画布当作**父→子**。**索引节点的自动目录**、**思维导图的层级**都靠这个方向沿连线推导。做有层级的结构（目录树、脑图、分类）时，**一定让 `from` 指父、`to` 指子**。
- 端点位置实时算最近边，你不用管锚点坐标。
- 图形 / 图片 / `textBox` 装饰文字框**不连线**。PDF/Markdown 附件当前可以连线、可以进入图谱，但仍不参与 Markdown 导出；AI 写笔记时通常只在“资料节点需要纳入关系图”时才主动给附件连线。

### 3.6 装饰（按需，锦上添花）

笔记一般不需要，但要"分区/标重点/配图"时可用。装饰节点**必须有 `width`/`height`**。

```json
// 带标题的分区盒子（底层，把相关节点圈在一起）
{ "id": "n_box", "kind": "shape", "shapeType": "group-box", "title": "第一章",
  "x": -40, "y": -40, "width": 800, "height": 400, "layer": "back" },
// 纯色色块（底层背景分区）
{ "id": "n_blk", "kind": "shape", "shapeType": "color-block",
  "x": 0, "y": 0, "width": 300, "height": 200, "layer": "back" }
```

- `group-box` 只是**视觉分区**，不会自动绑定框内节点；把要归类的节点坐标摆进它的范围即可。
- `layer`: `back`(底层) / `front`(顶层)。分区盒子/色块用 `back`。
- 图片需要资源文件（用户拖入更方便），AI 一般不直接写图片节点。

## 4. 正文 Markdown 语法大全（笔记的灵魂）

`body` 和便签/索引/预览/卡片的正文都用这套语法渲染。**代码节点除外**（整块当源码，不解析）。

### 4.1 基础块

| 写法 | 效果 |
|---|---|
| `# 标题` / `## 标题` | 大标题（h2） |
| `### 标题` | 中标题（h3） |
| `#### 标题` | 小标题（h4，再多 `#` 不会更大） |
| `- 项` / `* 项` / `+ 项` | 无序列表（**不支持嵌套**） |
| `1. 项` | 有序列表 |
| `> 引用` | 引用块 |
| `---` | 分隔线 |
| 空行 | 分段 |

> ⚠️ **不支持嵌套列表、行内图片**。需要层级请用「标题+列表」或多个节点+连线表达。

### 4.2 行内强调

| 写法 | 效果 |
|---|---|
| `**加粗**` / `__加粗__` | **加粗** |
| `*斜体*` / `_斜体_` | *斜体* |
| `~~删除线~~` | ~~删除线~~ |
| `` `行内代码` `` | 等宽代码 |
| `[文字](https://…)` | 链接（网址自动识别，裸 URL 也行） |

### 4.3 三件套：高光 / 字色 / 字号（画布特色，可叠加）

| 写法 | 效果 |
|---|---|
| `==文字==` | 黄色高光 |
| `{hl:blue\|文字}` | 指定色高光：`yellow`黄 `blue`蓝 `green`绿 `red`红 `purple`紫 |
| `{tc:red\|文字}` | 字色：`red`红 `blue`蓝 `green`绿 `orange`橙 `purple`紫 |
| `{fs:sm\|文字}` / `{fs:lg\|文字}` / `{fs:xl\|文字}` | 字号：小 / 大 / 特大 |

> 三者可嵌套叠加，如 `{hl:red|{tc:blue|{fs:lg|重点}}}`。**这是"内容色"，不受 UI 禁蓝限制**——高光/字色用蓝完全可以。但仍要克制，靠颜色编码语义（如统一"红=易错点、蓝=定义"），别花。

### 4.4 数学公式（MathJax，离线可用）

- 行内：`$...$`，如 `质能方程 $E=mc^2$`。
- 块级（独占一行、居中）：`$$...$$`。
- 公式跳转：同一正文里给公式打 `\label{eq1}`，用 `\eqref{eq1}` 引用，点击会跳转并闪烁。

### 4.5 Callout 提示框（Obsidian 风格，强烈推荐用于笔记）

```markdown
> [!tip] 解题技巧
> 先判断函数奇偶性，能省一半计算。

> [!warning] 易错
> 别忘了讨论分母为零的情况。
```

语法 `> [!类型] 可选标题`，下面接 `> 正文行`。**支持的类型**：

| 类型 | 语义 | 别名（也可写） |
|---|---|---|
| `note` | 笔记 | |
| `info` | 信息 | `todo` `infobox` |
| `tip` | 提示 | `hint` `important` |
| `abstract` | 摘要 | `summary` `tldr` |
| `success` | 成功/正确 | `check` `done` |
| `question` | 疑问 | `faq` `help` |
| `warning` | 注意 | `caution` `attention` |
| `danger` | 危险/错误 | `fail` `failure` `missing` `error` |
| `quote` | 引用 | `cite` |
| `example` | 示例 | |
| `bug` | 缺陷 | |

> Callout 是给笔记加层次、标重点的利器：定义用 `note`、技巧用 `tip`、易错用 `warning`、结论用 `success`、反例用 `danger`。

### 4.6 推导链 ` ```derive `（数学/物理推导杀手锏）

把多步推导渲染成**竖排步骤**（左侧步号、中间公式、右侧说明、步骤间 ↓ 连接）。每行格式：`公式 || 说明`（分隔符用 `||` 或 `‖`，说明可省）。

````markdown
```derive
f(x) = x^2 || 原函数
f'(x) = \lim_{h\to0}\frac{(x+h)^2-x^2}{h} || 代入定义
= \lim_{h\to0}(2x+h) || 展开约分
= 2x || 取极限
```
````

> 中间的公式部分是 LaTeX（会用 `\displaystyle` 行内排版），**不用写 `$`**。这是 STEM 笔记最该用的结构，比纯文字罗列清楚得多。

### 4.7 围栏代码块（正文里嵌代码）

````markdown
```python
def f(x):
    return x ** 2
```
````

- 支持语法高亮的语言：c/cpp/java/js/ts/py/python/go/rust/cs/php/swift/kotlin/matlab… 写 `text` 或不写语言则只等宽不上色。
- 注意区分：**正文里的 ` ``` ` 代码块** vs **整个代码节点 `kind:"code"`**。整段是代码就用代码节点；笔记里夹一小段代码就用围栏。

### 4.8 表格

```markdown
| 函数 | 导数 |
|---|---|
| $x^n$ | $nx^{n-1}$ |
| $\sin x$ | $\cos x$ |
| $e^x$ | $e^x$ |
```

对齐：表头分隔行用 `:---`(左) / `:---:`(中) / `---:`(右)。

### 4.9 双链 `[[名字]]`

正文里写 `[[某节点标题]]` 或 `[[标题|显示别名]]` 渲染成画布内跳转链接（点击跳到同名节点）。
> 注意：**正文里的 `[[]]` 是渲染成可点链接**；而**路径①导入时文件顶部的 `[[]]` 是生成连线**——两个场景别混。在手写 `.canvas`（路径②）里，节点间的关系请用 `edges` 表达，正文 `[[]]` 仅作为可点跳转补充。

## 5. 排版与美学守则

用户的审美是 **Apple 式简洁高级**：干净、留白足、层级清楚、克制。把这些落到笔记上：

**布局**
- **一张画布讲一个主题**。主题中枢放中间（常用索引节点），子概念向外铺。
- 善用**网格/树形**摆位：同层节点对齐、等距；父在左/上，子在右/下。
- 留足白：宁可空，不要挤。相邻节点中心 ≥320px 横 / ≥200px 纵。
- 关系网状用路径①让它自动布局；层级树用路径②摆好父子连线 + 让用户「脑图整理」。

**颜色（红线在此）**
- ✅ 笔记**内容**里的高光/字色可以用任何颜色（含蓝），用来编码语义。
- ❌ 但要**克制**：一张图节点色 ≤3~4 种；颜色要有**统一约定**（红=易错、蓝=定义…），不是装饰。
- ❌ 别堆砌：不要每个节点都上色、不要五颜六色。大面积留白+少量重点色，才高级。

**层次工具优先级**（做笔记按这个顺序选）
1. **节点拆分**：一个概念一张卡片，用连线表达关系——优于把所有东西塞进一个长节点。
2. **索引节点**：给章节群做目录中枢。
3. **Callout**：正文里标定义/技巧/易错/结论。
4. **推导链**：数学物理多步推导。
5. **高光/字色**：句内点睛，最后才用、最少用。

**别做的事**
- ❌ 不要给正文节点写死 `width`/`height`。
- ❌ 不要堆装饰图形抢信息层级。
- ❌ 不要嵌套列表（不支持）；改用标题分层或拆节点。
- ❌ 不要为了"酷"牺牲清晰。

## 6. 思维导图怎么做

画布有**自动排版**，你不用算坐标：

1. 建好节点：中心主题一个，下面各级子主题各一个节点（卡片即可，**不写坐标也行/随便给个值**）。
2. 连好父子线：每条 `edge` 的 `from`=父、`to`=子（**方向别反**）。
3. 交付后告诉用户：选中中心节点 → 点顶栏「脑图整理」→ 选 **树形** 或 **放射状**，画布会以中心为锚把整棵树排好、自动补齐缺的连线。

> 也可以走路径①：每个 `.md` 是一个主题，子主题文件顶部 `[[父主题]]`，导入后力导向会自然形成放射结构（但那是无向力导向，不是严格树形；要严格树形层级就用上面的脑图整理）。

折叠：给某个父节点加 `"mindmapCollapsed": true`，它的后代默认收起（用户悬停可预览）。

## 7. 「整理/美化用户已有画布」的做法

当用户给你一张现成的 `.canvas` 让你整理：

1. **读**：解析他的 `.canvas` JSON，看清现有 `nodes`/`edges`、各节点 `kind` 和正文。
2. **保结构、改呈现**：可以做——重排坐标对齐成网格/树、把长卡片正文加 Callout/推导链/标题分层、统一配色约定、给章节群加索引节点或分区盒子、把零散卡片按主题连线归簇。
3. **保留 id**：尽量**沿用原节点/连线的 `id`**，只改坐标/样式/正文；新增的才用新 id。这样用户的视野记忆、批注（按 id 存）不会错乱。
4. **不丢数据**：别删除你不理解的字段（如 `ink` 手写、`waypoints` 拐点、附件节点），原样保留。
5. 写回同一个 `.canvas`（或新存一份让用户对比）。

> ⚠️ 覆盖用户已有文件前，**先告诉用户你要改哪张、改了什么**，让他确认（这是项目铁律）。稳妥起见可另存为 `原名-整理版.canvas`。

## 8. 交付：怎么让用户用上

写完文件后，用一句话教用户操作（用户是电脑新手，说简单点）：

- **路径①（一组 `.md`）**：
  > "我把笔记写成了一个文件夹的 .md 文件，放在 `<路径>`。打开画布 → 起步页点「导入」→ 选这个文件夹，就会自动生成排好版、配好色的画布。"
- **路径②（一个 `.canvas`）**：
  > "笔记画布已生成：`<路径>`。打开画布 → 起步页点「打开」→ 选这个文件就行。"
  > 建议把文件直接存进项目的 `canvases/` 文件夹，和用户其它画布在一起，最好找。

## 9. 交付前自检清单

**通用**
- [ ] 文件 UTF-8 编码。
- [ ] 内容符合用户要的主题、准确无误（尤其公式、代码、定义）。
- [ ] 配色克制、有语义约定；没有滥用颜色。

**路径①（`.md` 文件夹）**
- [ ] 文件夹扁平，只有 `.md`，无子文件夹/杂文件。
- [ ] 文件名 = 想要的节点标题，且互不重复。
- [ ] `[[链接]]` 都在文件**顶部连续行**、精确匹配某个文件名、无自链接。

**路径②（`.canvas`）**
- [ ] 是合法 JSON（能被 `JSON.parse`），有 `version/nodes/edges`。
- [ ] 所有节点 `id` 唯一；每条 `edge` 的 `from`/`to` 都指向**存在的**节点 id。
- [ ] 正文节点（card/index/preview/sticky/code）**没写** `width`/`height`；装饰节点**写了**。
- [ ] 有层级的连线方向 `from`=父、`to`=子。
- [ ] `body` 里换行 `\n`、反斜杠 `\\` 都按 JSON 正确转义。
- [ ] 代码节点的 `language` ∈ `c`/`python`/`matlab`；正文围栏代码语言拼写正确。

## 10. 速查附录

**节点 `kind`**：`card`(默认) / `index`(目录) / `preview`(悬停展开) / `sticky`(便签) / `code`(代码) / `shape`(装饰) / `image`(图片) / `pdf` / `md`(附件) / `textBox`(装饰文字，通常由画布工具产生)

**命名色**：`gray` `blue` `green` `yellow` `red` `purple`

**高光**：默认黄写 `==文字==`；指定颜色写 `{hl:blue|文字}`，颜色名可用 `yellow` `blue` `green` `red` `purple`

**字色**：写 `{tc:red|文字}`，颜色名可用 `red` `blue` `green` `orange` `purple`

**字号**：`sm` `lg` `xl` — 写 `{fs:lg|文字}`

**Callout**：`note` `info` `tip` `abstract` `success` `question` `warning` `danger` `quote` `example` `bug`（含别名，见 §4.5）

**连线 `curve`**：`bezier` `straight` `elbow` `rounded-elbow` `s-curve` `smooth`

**连线 `arrow`**：`none` `to` `from` `both`

**代码节点语言**：`c` `python` `matlab`
